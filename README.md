# DPCコーディング支援ツール（診断群分類14桁コード グルーパー）

医師記録（SOAP）／退院サマリを貼り付けると、**診療科 → 病名 → 分岐 → 診断群分類14桁コード・診療報酬点数** を導く、ブラウザ完結のDPCコーディング支援ツールです。

🔗 **公開アプリ（GitHub Pages）**: https://itsuki-yano.github.io/dpc-coder/

---

## ⚠️ 重要な免責事項

- 本ツールは **コーディングの補助・参考** を目的としたものです。SOAP/サマリからの病名・分岐の推定は **キーワード照合ベースの参考補助** であり、**最終判断は必ず人（医師・診療情報管理士等）が行ってください**。
- 算定・請求の確定前に、必ず厚生労働省の **公式「DPC電子点数表」および定義告示・通知の原本** で照合してください。
- 本ツールは医療行為・診療報酬請求の正確性を保証するものではありません。利用により生じたいかなる結果についても責任を負いません。

## 特長

- **記録ファースト**：診療科を選び、SOAP/退院サマリ（複数日付OK）を貼り付けると自動で整理・推定
- **病名推定**：診断（A）行の病名に加え、「**膿瘍**」「穿孔」等の**所見**や**術式名**から推定。医師の病名がICDに無い・非標準でも推定可能
- **病名コード（MEDIS標準病名マスター 約27,000件）に対応**：病名コード（病名管理番号）・病名で検索でき、選んだ病名をDPC傷病名へ自動橋渡し。記録の診断から代表病名コードも自動表示
- **ICD-10全病名（約5,900件）を検索可能**
- **分岐に「📝 カルテ記載例」注釈**：各選択肢にどんなカルテ記載が該当するか（処置名・薬剤名・略称・スコア）を表示
- **14桁コード＋点数（入院期間 I/II/III）＋在院日数**を表示、複数候補は点数・解釈を比較
- **AI解析（LM Studio / Qwen）連携**（任意）：ローカルLLMで記録から病名・手術・処置を抽出し、既存の推定エンジンに反映
- **完全オフライン・単一ファイル**でも動作（`dpc-coder-standalone.html` / `index.html`）

## 使い方

### A. すぐ使う（推奨）
- 公開URL https://itsuki-yano.github.io/dpc-coder/ を開く
- または `dpc-coder-standalone.html`（= `index.html`）をダウンロードしてブラウザで開く（データ内蔵・オフライン可）

### B. 分割版
- `dpc-coder.html` と `dpc-data.js` を**同じフォルダ**に置いてHTMLを開く

### 操作の流れ
1. **診療科**を選択
2. **医師記録（SOAP）／退院サマリ**を貼り付け
3. 推定された**病名**を確認・選択（ICD-10検索も可）
4. **分岐**（手術・処置等1/2・副傷病・重症度）を確認（自動反映・記載例つき）
5. **結果**：14桁コード・点数・入院期間

## AI解析（LM Studio / Qwen）連携 ※任意

ローカルで動く LLM（[LM Studio](https://lmstudio.ai/) の OpenAI 互換サーバ）に接続し、自由記載の記録から**病名・手術・処置・副傷病・重症度**を抽出して、抽出結果を記録に追記 → 既存の病名推定・分岐推定・病名コード橋渡しに自動反映します。

### 使い方
1. LM Studio で **Qwen 系モデル**を読み込み、「**Local Server**」を起動（既定 `http://localhost:1234`）。
2. アプリの「② 記録入力」で **🤖 AIで解析（LM Studio / Qwen）** を開く。
3. エンドポイント（既定 `http://localhost:1234/v1/chat/completions`）とモデルを設定（「接続テスト」で自動取得・保存）。
4. 記録を貼り付けて「**この記録をAI解析**」→ 抽出結果が記録に反映され、病名・分岐が自動推定されます。

### ⚠️ 重要：ブラウザの制約
- **https（GitHub Pages）からは `http://localhost` を呼べません**（mixed content）。AI連携を使う場合は、**ローカルでファイルを直接開く**か、**`http://localhost` で配信**してください（`index.html` / `dpc-coder-standalone.html` をダウンロードして開けばOK）。
- LM Studio 側のサーバ起動・モデル読込・CORS 許可が前提です。
- AIの抽出はあくまで**補助**です。最終判断は人が行ってください。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `index.html` / `dpc-coder-standalone.html` | 単一ファイル版（データ内蔵・Pages公開） |
| `dpc-coder.html` + `dpc-data.js` + `dpc-byomei-master.js` | 分割版（軽量HTML＋DPCデータ＋病名コード辞書） |
| `dpc-byomei-master.js` | MEDIS標準病名マスター（病名コード辞書・約27,000件） |
| `dpc-sample-records.md` | 動作確認用の架空サンプル記録（退院サマリ5・医師記録10） |

## データ出典

- 厚生労働省「**DPC電子点数表**」（2026年5月19日版）の公開データ（傷病名・ICD-10・手術等分岐・点数・在院日数）を解析して使用。
- 病名コードは **MEDIS-DC「ICD10対応標準病名マスター」Ver.5.18（履歴版）** の病名管理番号を使用（病名コード↔病名）。※履歴版にICD-10列は無いため、DPCへの紐付けは病名名称ベースで行っています。
- ICDは **ICD-10準拠**です（DPC公式データはICD-10ベースのため、ICD-11コードは未収録）。

## 技術

- 依存ライブラリなしの **素のHTML/CSS/JavaScript**。ビルド不要。
- データは公式Excelから `dpc-data.js`（`window.DPC_DATA`）として生成。
